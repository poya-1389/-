# درخواست آپدیت و توسعه Nova Self

این آپدیت باید بدون ایجاد تداخل با قابلیت‌های فعلی پروژه پیاده‌سازی شود. قبل از تغییرات، سیستم فعلی پنل، Callbackها، Permissionها، Session Management و دیتابیس بررسی شود.

هدف این نسخه:

* افزایش کنترل‌پذیری Self
* بهبود رابط کاربری
* رفع مشکلات پنل ادمین
* ایجاد پنل قابل استفاده در تمام چت‌ها
* جلوگیری از دسترسی غیرمجاز کاربران دیگر

تمام تغییرات باید با Clean Code، مدیریت خطا، امنیت و ذخیره‌سازی صحیح در دیتابیس انجام شوند.

---

# 1. رفع مشکل تغییر وضعیت کاربر از پنل ادمین

در حال حاضر وقتی ادمین از پنل مدیریت وضعیت یک کاربر را تغییر می‌دهد:

* تغییر اعمال نمی‌شود.
* مقدار قبلی باقی می‌ماند.

این مشکل کامل بررسی و رفع شود.

موارد بررسی:

* Queryهای دیتابیس
* Update شدن مقدار صحیح
* Refresh شدن اطلاعات پنل
* Cache احتمالی
* هماهنگی دیتابیس با Session کاربر

بعد از تغییر وضعیت توسط ادمین:

* مقدار جدید بلافاصله اعمال شود.
* در پنل کاربر نیز مقدار جدید نمایش داده شود.

---

# 2. تغییر رنگ دکمه‌های بازگشت

تمامی دکمه‌هایی که عملکرد زیر را دارند:

* بازگشت
* لغو
* انصراف
* برگشت به منوی قبل

باید:

رنگ قرمز داشته باشند.

نام تمام این دکمه‌ها تغییر کند به:

```
➜ بازگشت
```

هیچ نام دیگری برای این نوع دکمه‌ها استفاده نشود.

---

# 3. رنگ‌بندی کامل دکمه‌ها

تمام دکمه‌های منوها بررسی شوند.

دکمه‌هایی که بدون رنگ هستند باید رنگ مناسب دریافت کنند.

قوانین:

* فعال بودن قابلیت → سبز
* غیرفعال بودن قابلیت → قرمز
* عملیات مثبت → سبز
* بازگشت و لغو → قرمز
* اطلاعات و حساب کاربری → آبی
* عملیات مهم یا تایید → رنگ مناسب اختصاصی

تمام منوها باید از سیستم رنگ‌بندی یکسان استفاده کنند.

---

# 4. تغییر سیستم منشی

قابلیت منشی باید مانند سایر قابلیت‌های Self باشد.

به جای چند دکمه:

فقط یک دکمه نمایش داده شود.

مثال:

```
منشی ✓
```

یا

```
منشی ✕
```

عملکرد:

* روشن → سبز
* خاموش → قرمز

تمام تنظیمات منشی داخل همان بخش مدیریت شود.

---

# 5. تغییر رنگ انتقال الماس

در بخش حساب کاربری:

دکمه:

```
انتقال الماس
```

رنگ سبز داشته باشد.

---

# 6. بهبود قابلیت بازیابی Session

دکمه بازیابی نشست باید قابلیت لغو داشته باشد.

قبل از انجام عملیات:

مانند حساب کاربری، یک تاییدیه نمایش داده شود.

مثال:

```
آیا مطمئن هستید که می‌خواهید نشست خود را بازیابی کنید؟

⚠️ اطلاعات حساب حذف نمی‌شود اما نیاز به ورود دوباره دارید.
```

دکمه‌ها:

```
🟢 تایید
🔴 ➜ بازگشت
```

اگر کاربر بازگشت را بزند هیچ تغییری انجام نشود.

---

# 7. افزایش قابلیت‌های پنل ادمین

ادمین باید کنترل بیشتری روی Self کاربران داشته باشد.

موارد قابل تغییر توسط ادمین:

## فونت ساعت

* تغییر فونت
* خاموش کردن قابلیت

## فونت تاریخ

* تغییر فونت
* خاموش کردن قابلیت

## حالت متن

* تغییر حالت فعال
* خاموش کردن
* انتخاب نوع متن

## Action ها

* فعال/غیرفعال کردن
* تغییر تنظیمات

## سایر قابلیت‌های Self

تمام قابلیت‌هایی که کاربر از پنل شخصی مدیریت می‌کند، در صورت نیاز از پنل ادمین نیز قابل کنترل باشند.

---

# 8. ایجاد پنل Self در تمام چت‌ها

پنل Self فقط محدود به پیوی ربات نباشد.

کاربر بتواند در هر نوع چتی پنل ایجاد کند:

* گروه‌ها
* پیوی کاربران
* پیوی ربات‌ها
* Saved Messages

---

# 9. دستور ایجاد پنل

دستور:

```
.پنل
```

وقتی توسط صاحب Self ارسال شد:

در همان چت یک پنل شیشه‌ای ایجاد شود.

عملکرد پنل باید مشابه پنل موجود در پیوی ربات Nova باشد.

تمام تنظیمات Self از همان پنل قابل تغییر باشند.

---

# 10. سیستم Permission پنل

امنیت بسیار مهم است.

فقط شخصی که Self روی اکانت او فعال است اجازه مدیریت دارد.

یعنی:

اگر کاربر A صاحب Self باشد:

* فقط A می‌تواند دکمه‌های پنل را استفاده کند.

افراد دیگر:

* نتوانند تنظیمات را تغییر دهند.
* با کلیک روی دکمه‌ها پیام خطای دسترسی دریافت کنند.

---

# 11. بستن پنل

در پایین تمام منوهای پنل:

یک دکمه اضافه شود:

```
✕ بستن پنل
```

رنگ:

قرمز

عملکرد:

وقتی کاربر کلیک کرد:

* همان پنلی که در آن چت ایجاد شده حذف شود.
* پیام‌های اضافی ایجاد نشود.
* فقط صاحب Self بتواند پنل را ببندد.

---

# 12. مدیریت چند پنل همزمان

بررسی شود که:

* یک کاربر بتواند در چند چت پنل داشته باشد.
* پنل‌ها با هم تداخل نداشته باشند.
* Callback یک پنل روی پنل دیگر اثر نگذارد.
* اطلاعات هر پنل مربوط به همان چت باشد.
